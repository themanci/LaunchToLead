Rally Agile User Story Template

Initiative: Standardized Requirements Documentation

Adoption: Used by 50+ team members across multiple product lines

User Story US12345
Sprint 23.5 | 5 points
Story Title
Implement Real-Time Seeding Rate Adjustment Based on Soil Conditions
As a Precision Agriculture Engineer
I want the seeding system to adjust planting density in real-time
So that seed placement optimizes for varying soil moisture and compaction levels
Acceptance Criteria
✓ Given soil moisture sensor reads below 25% water content
When planter enters low-moisture zone
Then seeding rate increases by 8-12% within 2 seconds
✓ Given soil compaction sensor detects hardpan layer
When compaction exceeds 300 PSI
Then system reduces seeding rate by 5-10% to prevent seed bounce
✓ Given adjustment exceeds ±15% from target rate
When operator is in auto-mode
Then display shows warning notification requiring manual confirmation
Technical Implementation Notes
• Integrate with existing CAN bus sensor network (ISO 11783)
• Use PID controller algorithm for smooth rate transitions
• Store adjustment history in NVRAM for post-season analysis
• Unit test coverage required for all edge cases (sensor failures, out-of-range values)
• HIL testing required on 8R Series test bench before release
Definition of Done
✓ Code reviewed by senior engineer
✓ Unit tests passing (≥85% coverage)
✓ HIL tests completed - zero defects
✓ Documentation updated in Confluence
✓ Product Owner acceptance obtained
✓ Merged to main branch
Owner: M. Manci
Created: 04/12/2023
