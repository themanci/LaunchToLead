# Impact Bullet Builder Framework

> **Purpose:** This file defines the resume bullet analysis framework used by LaunchToLead. Upload this as a context file in Claude Projects alongside the audit protocol.

---

## The Formula

Every strong resume bullet includes these 4 components:

```
Accomplishment + Metric + How + Why = IMPACT
```

| Component          | What It Answers                                | Example Fragment                                           |
| ------------------ | ---------------------------------------------- | ---------------------------------------------------------- |
| **Accomplishment** | What did you DO? (not "responsible for")        | Redesigned the thermal monitoring pipeline...              |
| **Metric**         | How much? How many? What %? What $?             | ...reducing false alarms by 74%...                         |
| **How**            | What tools, methods, or skills did you use?     | ...using Python-based anomaly detection in MATLAB...       |
| **Why**            | Why does the CEO care? Business impact.          | ...saving $1.2M/year in unnecessary shutdowns.             |

---

## Three Tiers of Value

Every bullet falls into one of three tiers. The goal is to push every bullet toward ★★★.

### ★ You Did What Was Asked
- Describes a duty or responsibility, not an accomplishment.
- No metric, no method, no outcome.
- Every engineer with the same job title could write this exact bullet.
- Example: *"Maintained spreadsheets"*
- Example: *"Responsible for testing software modules"*
- **Resume impact:** Invisible to hiring managers. Gets skipped.

### ★★ You Went Above and Beyond
- A real accomplishment — you saw a problem and fixed it, or exceeded expectations.
- Has some numbers or specifics, but missing full impact chain.
- Example: *"Built an automated tracking system for 3 major programs, replacing a manual process that took 4 hours per week"*
- **Resume impact:** Gets considered. Better than most, but not memorable.

### ★★★ You Spread the Impact Far and Wide
- Full formula: Accomplishment + Metric + How + Why.
- Shows the value reaching beyond you — trained others, adopted across teams, affected the business.
- Example: *"Built an automated tracking system for 3 major programs, saving 85% of manual reporting time, then trained 4 team leads to use it — now adopted across 3 departments for daily executive visibility"*
- **Resume impact:** Gets hired. This is the target for every bullet.

## Six Key Principles

### 1. Personal Ownership
- Flag any "we/team/group/our" language.
- Replace with the candidate's specific personal contribution.
- "Assisted with" and "Contributed to" hide the candidate's role.
- **Test:** If you removed the candidate from the team, would this bullet still be true? If yes, it's not showing personal ownership.

**Before → After:**
- ✗ "We delivered the project on time" → ✓ "I led sprint planning and dependency tracking for 3 workstreams"
- ✗ "The team improved system reliability" → ✓ "I designed the automated monitoring pipeline"
- ✗ "Our group redesigned the workflow" → ✓ "I proposed and implemented the new process flow"

### 2. Problem Context / "So What?"
- Paint the pain or problem FIRST, then show the accomplishment.
- The bigger the context, the bigger the impact feels.
- Three ways to add context:
  - **Stakes:** "On a $4.2M project with zero margin for delay..."
  - **Before/After:** "Reduced from 12 hours to 45 minutes..."
  - **Scale:** "Across 14 manufacturing sites..."

### 3. Accomplishments vs. Duties
- A duty describes what you were assigned. An accomplishment describes what changed because you were there.
- **Quick filter:** If the bullet could appear on anyone else's resume who had the same job title, it's a duty — not an accomplishment.
- "Managed project timelines" = duty. "Accelerated delivery by 3 weeks by restructuring the sprint backlog" = accomplishment.

### 4. BS Meter
- No vague claims without proof.
- "Excellent communicator" means nothing. Show it: "Presented weekly status updates to 4 VPs, securing $200K in additional budget."
- Vague superlatives without evidence should be removed or replaced with specifics.

### 5. Action Verbs
- **Never start a bullet with:** "Responsible for," "Duties included," "Assisted with," "Helped with," "Contributed to," "Supported," "Involved in," "Participated in."
- **Use strong verbs:** Engineered, Designed, Built, Automated, Reduced, Accelerated, Architected, Led, Shipped, Deployed, Optimized, Eliminated, Migrated, Implemented.

### 6. Skills in Context
- Don't just list tools in a Skills section. Show them IN the bullets.
- "Python" in a skills list = weak. "Built a Python automation script that reduced manual reporting by 85%" = strong.
- **Test:** If you removed the Skills section entirely, could a reader still tell what tools you know? If not, the bullets need more "How."

## The "So What?" Chain

For every accomplishment, keep asking "So what?" until you hit something a VP would care about. Usually takes 2–3 levels:

- **Level 1:** "I automated the weekly report." — *So what?*
- **Level 2:** "It saved 4 hours per week for 3 people." — *So what?*
- **Level 3:** "Those engineers were freed up to focus on the product launch, which shipped 2 weeks ahead of schedule."

Level 3 is your "Why." That's what goes on the resume.

## Finding Hidden Metrics

Five sources for metrics candidates think they don't have:

1. **Ask your manager or team lead** — "What was the business impact of [project]?" They often have data you don't.
2. **Check company dashboards & reports** — Quarterly reports, Jira stats, project retrospectives, internal KPI dashboards.
3. **Calculate time savings** — Time saved per day × people affected × 250 working days/year × hourly rate = annual dollar value.
4. **Use conservative estimates with qualifiers** — "Reduced processing time by ~30%" is better than no number at all.
5. **Count scope and scale** — "Served 500+ users" or "Deployed across 3 facilities" — scale is a metric too.

### Time Savings Calculator
```
Time saved/day × People affected × 250 working days/year = Hours saved/year
Hours saved/year × $50/hr (or actual rate) = Dollar value
```
Example: 30 min/day × 5 people × 250 days = 625 hours/year × $50 = **$31,250 saved annually**

### Common Metrics by Category

| Time & Efficiency | Quality & Reliability | Cost & Revenue | Scale & Scope |
|---|---|---|---|
| Hours saved/week | Defect/error rate reduction | Cost savings ($) | Users/customers served |
| Cycle time reduction % | Uptime/availability % | Revenue generated | Team size managed |
| Projects ahead of schedule | Customer satisfaction | Budget managed | Systems/facilities impacted |

## What Counts as "How"

Four categories:

1. **Tools & Technologies** — SolidWorks, Python, MATLAB, Altium, Jenkins, SAP, Kafka, Spark, Terraform, etc. Name the specific tool.
2. **Methodologies & Processes** — Lean, Six Sigma, Agile, DMAIC, root cause analysis, DOE, CI/CD, TDD, etc.
3. **Collaboration & Leadership** — Cross-functional coordination, stakeholder reviews, mentoring, leading design reviews.
4. **Problem-Solving Approaches** — Prototyping, testing, debugging, reverse engineering, thermal imaging diagnosis, A/B testing.

## Common Impact Categories for "Why"

| Category | Examples |
|---|---|
| **Cost Savings** | "Eliminating $45K in annual rework costs" · "Reducing scrap rate, saving $120K/year" |
| **Time Savings** | "Freeing 3 engineers for higher-priority work" · "Cutting review cycle from 2 weeks to 3 days" |
| **Risk Reduction** | "Preventing SLA violations on a $200K contract" · "Eliminating a single point of failure" |
| **Scale & Reach** | "Enabling on-time delivery for 12 clients" · "Now adopted across 3 departments" |

## Example ★★★ Bullets by Discipline

**Mechanical:** "Redesigned conveyor belt alignment system using SolidWorks FEA analysis, reducing assembly line downtime by 40% and saving $85K in annual maintenance costs — during a quarter when unplanned stops were costing $12K/hour"

**Software:** "Architected a microservices migration using Kubernetes and Terraform, reducing deployment time from 4 hours to 12 minutes and enabling the team to ship features 3x faster — directly supporting a product launch that added $2M ARR"

**Data:** "Engineered a real-time patient monitoring pipeline processing 10K vital-sign readings per second using Kafka and Spark Streaming, triggering automated SNS alerts that reduced clinician response time from 8 minutes to 90 seconds across 200 monitored patients"

**Electrical:** "Designed a custom PCB power distribution board using Altium Designer, reducing voltage ripple by 60% and eliminating field failures that were costing $18K/month in warranty claims across 3 product lines"
