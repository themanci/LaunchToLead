Traffic Routing — Who Sees What

Traffic Source How They Arrive What Happens
LinkedIn Ads URL has utm_content=graduating-soon (or early-engineers, recently-graduated) 50/50 split: v1 vs matching variant (v2, v3, or v4)
Unpaid Traffic No utm_content — LinkedIn posts, Skool, Facebook, direct, SEO, etc. 50/50 split: v1 vs v5
Testing / Preview URL has ?variant=v5 (or v1, v2, v3, v4) Forces that variant — bypasses all split logic
Returning Visitor Has ltl_variant in localStorage Locked to their first assignment forever

50/50 Split Mechanics

Each audience segment has its own counter stored in localStorage. The counter increments on every new visitor. Odd = v1 (control), Even = variant.

This guarantees a perfect 50/50 at every even number of visitors — critical at low traffic volumes.

Counter Key Segment Odd Count → Even Count →
ltl_counter_graduating-soon Paid — students v1 v2
ltl_counter_early-engineers Paid — early career v1 v3
ltl_counter_recently-graduated Paid — recent grads v1 v4
ltl_counter_unpaid Unpaid — all engineers v1 v5

2

→ Use stored variant. Locked forever.

3

URL has utm_content? (paid ads)

→ Map to audience variant (v2/v3/v4), 50/50 split vs v1.

4

No utm_content? (unpaid traffic)

→ 50/50 split: v1 vs v5. Counter key: ltl_counter_unpaid.
